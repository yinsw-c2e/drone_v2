# 全页面生产级 UI / UX 审计

本轮审计以 `docs/EXECUTION.md` §13「低空指挥中心」为唯一视觉目标，UI UX Pro Max 仅作为触控、对比度、空/错/加载态、响应式和可访问性清单参考。所有新增样式继续取 `src/styles/tokens.scss`。

## 共享骨架

- `PageHeader`：统一页面标题、业务说明和角色标识，减少各页散写标题导致的信息层级不一致。
- `SectionHeader`：统一区块标题、说明和右侧轻操作，避免区块只放一个孤立标题。
- `NoticeBar`：统一页面内错误、阻断、生产化能力边界和成功反馈，避免按钮点击无解释或技术错误外露。
- `ActionCard` / `wd-card`：统一首页最强主行动，发单、接单、调度入口都按 §13.9 的首屏 CTA 强度处理。
- `InfoCell`：以 `wd-cell` 为外壳，显式渲染标题、说明和值，解决 H5/小程序下 wot cell title slot/props 可见性不稳定的问题。
- `RouteHero`：低空航线视觉锚点只用于匹配、追踪、飞手任务等真实航线页，承载起终点、空域、飞行器位置、ETA、距离和电量；首页和后台不再重复复用假地图。
- `RouteHero` 指标契约：本轮收口为显式 `metrics` 业务标签。默认只用于真实航线指标“预计到达 / 航线距离 / 电量”，首页、资产、通知、分账和合规场景必须传入自定义中文业务指标，避免“设备资产/距离”“T+7/电量”等错配。
- `KpiStrip`：本轮新增紧凑横向指标，替代大面积等权白卡，应用于首页、追踪、任务、保险和钱包。
- `IconActionGrid`：本轮新增 icon/symbol + label + status 入口，用于认证、信用、保险、设备、调度、钱包等次级动作。
- `StageStepper` / `CommandPanel`：本轮新增迷你阶段轨道和指挥动作面板，让主推进、应急、终态阻断拥有明确业务说明。
- `AdminDataPanel`：后台面板骨架改为 `wd-card` 外壳，统一桌面/移动的表格队列、报表、审计日志和操作位。
- `wot-design-uni` 接管：登录、业主、飞手、机主和后台关键动作均使用 `wd-navbar`、`wd-card`、`wd-cell-group`、`wd-cell`、`wd-tag`、`wd-notice-bar`、`wd-steps`、`wd-button`、`wd-action-sheet`、`wd-checkbox`；分段切换用 wot 按钮组保证移动端可见文本稳定。
- `display-labels`：统一角色、货物、认证、设备、运力、保单、理赔和流水状态中文展示，样式仍可使用枚举 class，但用户不再看到 raw status。
- `auditLogDetailLabel`：后台审计日志对历史详情做兼容清洗，旧的 `Mock/provider/escrow/valuable` 等内容展示为中文业务文案。
- `claimLiabilityLabel` / `orderDisplayTitle` / `capacityHeatmapLabel`：后台理赔责任、订单标题 fallback 和报表热力点统一转为业务文案，旧存储里的 `Mock`、裸订单号和裸运力编号不再直出到页面。
- `ProSegmentedControl`：按 `ui-ux-pro-max` 触控与分段控件规则新增，等宽段、清晰选中态、触控目标达标，替代匹配页原先挤压的胶囊按钮。
- `MatchCandidateCard`：按 Logistics/Delivery 报价列表重构，结构固定为“设备/飞手身份、报价、ETA/距离/综合分、承运保障、推荐理由、选中状态”，移除彩色标签泛滥。
- 全局样式补充按钮按压/禁用、`metric-grid`、`list-stack`、`admin-page/admin-metrics`，保持移动端触控和后台桌面密度。
- 本轮 UI UX Pro Max 复核：补 `docs/ui-redesign/UI_UX_PRO_MAX_PLAN.md`，明确产品类型、风格、组件规范、页面矩阵和验收清单；真实页面不引入第二套 UI 框架，只以 skill 作为审计和结构参考。
- 后台移动动作区：订单、认证、风控和理赔按钮在移动端分行网格展示，按钮文字 `white-space: nowrap`，避免中文动作被压成竖排。
- 第二轮收口：清除面向用户页面的英文模板眉标和工程实现词，所有页面改为中文业务语义；后台桌面宽度补左侧运营导航和表格式信息密度。
- 本轮桌面密度收口：后台不再沿用移动 `rpx` 窄容器，1280 H5 视口下 `.admin-page` 为 1280 宽、主区约 1076 宽、订单管理约 1046 宽；订单管理全宽表格式，认证和报表保留运营队列密度。
- 本轮首页工作台收口：业主首页改为调度任务板，机主首页改为资产运力池，不再以 `wd-card + InfoCell` 示例结构作为首屏主体。
- 本轮截图策略收口：匹配、追踪、飞手任务等 fixed 底栏页使用正常移动视口截图；匹配页滚动到候选列表位置，避免 full-page fixed 底栏伪影被误判为遮挡。
- 匹配状态保护：匹配页动作规划带入订单状态。只有 `Matching` 订单展示候选和“确认下单”；执行中、已完成、已结算、异常和取消订单会降级为“查看追踪 / 查看结算 / 重新发单”，并显示业务解释。
- 后台端到端反馈：跑通面板将“当前可执行性说明”和“本次操作结果”拆成独立状态；成功结果优先显示，不再因为跑通后运力被占用而同时展示“无在线运力”阻断。

## 页面审计结果

## 仍需后续提升

- 当前地图仍是轻量 MapTrack 模拟视觉，生产级应接真实地图、禁飞区、轨迹回放和视频/遥测流。
- 当前图表仍以指标、热力点和报表行表达，生产级后台应接真实图表组件和可筛选时间维度。
- 当前 ConfirmSheet 尚未统一抽取，不可逆操作已尽量用业务阻断/反馈，但生产级可继续补二次确认组件。
